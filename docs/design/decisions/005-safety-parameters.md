# 安全制約パラメータ設計

## 概要

HOCBF-QPのクラスK関数パラメータ（γ_pos, γ_force）、ロバスト化方針、適応則パラメータ（α_relax, α_shrink, ε, ε_danger）の設計方針を定める。

## 設計判断

### 判断1: γ_pos / γ_force — 物理的直感で初期値、シミュレーションで調整

**問題**: HOCBF条件のクラスK関数パラメータ γ_pos, γ_force をどう決めるか。u_lb, u_ub の式に直接影響し、制約の「攻撃性」を決定する。

**選択肢**:
1. 物理的直感で初期値を設定し、シミュレーションで調整
2. システムの固有振動数や制御帯域から理論的に導出
3. グリッドサーチで網羅的に探索

**決定**: 選択肢1 — 物理的直感で初期値、シミュレーションで調整

**理由**:
- γ が大きいほど制約が攻撃的（安全境界から離れていても早めに制御介入）
- γ が小さいほど制約が寛容（境界ギリギリまで名目制御を許す）
- 1-DOFでは u_lb, u_ub の式が解析的なので、γ の効果を直接計算で確認できる
- 最終的にはシミュレーション結果（安全集合の維持、応答の滑らかさ、G(t)の余裕）を見て判断する

**初期値の指針**:

γ の物理的意味: HOCBF の補助関数 ψ₁ = ψ̇₀ + γψ₀ において、γ はψ₀ の減衰率を規定する。ψ₀ = h(x) なので、γ ≈ 「安全余裕がゼロに向かう速度の許容上限」。

```
γ が大きすぎ → u_ub - u_lb が狭くなりすぎ → G < 0 で feasibility 破綻
γ が小さすぎ → 安全境界に近づいても制約が効かない → 制約違反のリスク
```

初期値の目安:
```
γ_pos   = 20 [rad/s]  （変形防止側）
γ_force = 20 [rad/s]  （滑落防止側）
```

根拠:
- 系の応答時間: 1-DOFハンド（M=0.000054 kg·m²）は非常に軽量で高速応答
- DOBのカットオフ: g_dis = 600 rad/s。γ はこれより十分小さくあるべき
- 経験則: γ ≈ 制御帯域の 1/10〜1/5 程度が安定しやすい（600/30 ≈ 20）

**チューニング手順**:
1. γ_pos = γ_force = 20 で基本シミュレーション
2. G(t) = u_ub - u_lb の最小値を確認。G_min < 0 なら γ を下げる
3. 制約違反（h_lb < 0 or h_ub < 0）があれば γ を上げる
4. 応答の滑らかさ（u* の急変がないか）を確認

**トレードオフ**:
- **利点**: 少ない試行で妥当な値に収束できる、物理的解釈が明確
- **欠点**: 最適値の保証はない（だが研究の目的は安全性の検証であり最適性ではない）

### 判断2: ロバスト化 — まず推定誤差なしで動作確認

**問題**: RFOB推定誤差 d(t) のロバスト化をシミュレーションでどこまで実装するか

**選択肢**:
1. まず d(t)=0（推定誤差なし）で基本動作確認。後でロバスト版を追加
2. 最初からロバストバリア関数 h̃_i を実装

**決定**: 選択肢1 — まず推定誤差なしで動作確認

**理由**:
- step1_formulation.md の理論導出も d(t)=0 を先に導き、条件(iii)で補足している
- 推定誤差なしで安全集合の前方不変性を確認するのが最優先
- ロバスト化は u_lb, u_ub に ±M×d_max のマージンを追加するだけなので、後付けが容易
- novelty_theorems.md の定理1（ISSf-HOCBF安全性）の検証は、基本版が動いてから

**段階的な実装計画**:
```
Phase A: d(t)=0、推定誤差なし
  - K̂_obj = K_obj（完璧な推定）
  - 標準 HOCBF-QP の動作検証
  - γ_pos, γ_force のチューニング

Phase B: 推定遅延あり
  - RFOB の LPF による遅延を再現
  - K̂_obj ≠ K_obj だが有界（d_max は未使用）

Phase C: ロバスト版
  - d_max マージンを u_lb, u_ub に追加
  - ロバストバリア関数 h̃_i の前方不変性を検証
```

**トレードオフ**:
- **利点**: デバッグが容易、段階的に複雑さを増やせる
- **欠点**: Phase A の結果は理想的すぎて論文には使えない（Phase B 以降のデータを使う）

### 判断3: 適応則パラメータ — 物理的制約から導出

**問題**: θ_max 適応則のパラメータ（α_relax, α_shrink, ε, ε_danger, θ_max_0, θ_max_min, θ_max_ceiling）をどう決めるか

**選択肢**:
1. 物理的制約から導出
2. シミュレーションで広い範囲を探索

**決定**: 選択肢1 — 物理的制約から導出

**理由と具体的な導出**:

#### α_shrink（縮小速度）の上界

step1_formulation.md の安全条件(i)より:
```
α_shrink(t) ≤ [u_ub_static(t) - u_lb(t)] / [M × 2γ_pos]
```

u_ub_static は θ̇_max = 0 としたときの u_ub。feasibility 余裕が大きいほど速く縮小できる。

初期値の目安:
```
α_shrink_nominal = 0.5 [rad/s]
```
理由: θ_max の典型的な範囲が 0.01〜0.10 rad なので、0.5 rad/s なら 0.02秒で 0.01 rad 縮小。十分に速いが急すぎない。

実装では動的に上界を計算し、α_shrink_nominal と min を取る。

#### α_relax（緩和速度）

物理的制約なし（安全集合が拡大する方向なので危険ではない）。ただし速すぎると K̂_obj が安定する前に θ_max が大きくなりすぎる。

```
α_relax = 0.1 [rad/s]
```
理由: α_shrink の 1/5 程度。「縮小は速く、緩和は慎重に」の設計思想。

#### ε（安定判定閾値）

RFOB の K̂_obj 推定のノイズレベルに基づく。
```
ε = 推定ノイズの標準偏差 × 3
```

シミュレーション Phase A（K̂_obj = K_obj）では ε = 0.1 [Nm/rad/s] 程度から始める。
Phase B（RFOB あり）で RFOB のノイズレベルを測定し調整。

#### ε_danger（急変検出閾値）

K̂_obj が明確に低下している判定。ε よりも大きく設定。
```
ε_danger = ε × 5 = 0.5 [Nm/rad/s]
```

#### θ_max_0（初期値）

最も壊れやすい物体（紙コップ空: θ_yield = 0.01 rad）でも安全な値。
```
θ_max_0 = 0.005 [rad]
```
理由: 紙コップの θ_yield の半分。全物体共通。

#### θ_max_min / θ_max_ceiling

```
θ_max_min     = 0.002 [rad]   （これ以下には縮小しない）
θ_max_ceiling = 0.10  [rad]   （これ以上には緩和しない）
```

**トレードオフ**:
- **利点**: 各パラメータに物理的根拠がある、設計指針として論文に記載可能
- **欠点**: 概算ベースなので微調整は必要
