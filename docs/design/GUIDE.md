# 設計書作成ガイド

## 目的

hocbf_grasp_sim の設計判断を記録し、将来の自分・指導教員が「なぜこう作ったか」を追えるようにする。

**対象読者**: 著者自身（榎本爽我）、指導教員（中村先生）

**実装ステータスに関する原則**: 設計書に「未実装」「実装済み」等の時間依存の記述を書かない。実装状況は別途管理する。

## ディレクトリ構成

```
design/
├── GUIDE.md          # 本ファイル
└── decisions/        # アーキテクチャ決定記録（ADR）
    └── 001-technology-stack.md
```

## ADR（Architecture Decision Record）

技術選定やアーキテクチャ判断を記録する。

### ファイル命名

```
NNN-kebab-case-title.md
```

### 必須セクション

| セクション | 内容 |
|-----------|------|
| 概要 | 何を決定するか |
| 設計判断 | 問題・選択肢・決定・理由・トレードオフ |
| 関連ドキュメント | 参照先 |

### コード参照形式

```
`src/controller.cc:42` — HOCBF-QP の clamp 実装
```

## 品質チェックリスト

- [ ] 全セクションが埋まっている（空セクションなし）
- [ ] コード参照のパスが正しい
- [ ] 理論の数式参照がある場合、`~/lab/exp/` 内の対応文書を明記
